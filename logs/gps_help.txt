WWV-GPS Timing Verification Tool
Phoenix Nest MARS Suite - Phase 3 GPS Integration

Usage: D:\claude_sandbox\phoenix_sdr\bin\wwv_gps_verify.exe -t <timestamp> -o <offset_ms> [-p <port>] [-d]

Required:
  -t <timestamp>  Recording start time (ISO 8601 or Unix epoch)
                  Examples: 2025-12-13T13:15:30 or 1734095730.5
  -o <offset_ms>  WWV offset from file start to minute boundary (ms)
                  This is the "Offset from file start" from wwv_sync

Optional:
  -p <port>       GPS serial port (e.g., COM6 or /dev/ttyUSB0)
                  If provided, reads current GPS for live comparison
  -d              Decode mode: show expected vs actual minute
  -h              Show this help

Example workflow:
  1. Record WWV:    phoenix_sdr -c 10000000 -s 48000 -o wwv.iqr
  2. Note the time when you start recording
  3. Analyze:       wwv_sync wwv.iqr 0 300
  4. Verify:        wwv_gps_verify -t 2025-12-13T13:15:30 -o 21350.5

The tool calculates:
  - UTC time of each minute boundary in the recording
  - Expected vs actual minute (if BCD decode available)
  - Error vs GPS reference (if -p port specified)
